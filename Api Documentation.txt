Travel Planner API Documentation

Base URL
http://localhost:3000/api

===========================================

Authentication Endpoints

POST /auth/signup
Purpose: Register a new user account with profile information and optional admin role assignment.
Body: { username, email, password, profilePic?, firstName, lastName, phoneNumber, city, country, role? }

POST /auth/login
Purpose: Authenticate user credentials and return user session data including userId.
Body: { email, password }

===========================================

Trip Management Endpoints

POST /trip/create
Purpose: Create a new travel trip with destination, dates, group size, and description.
Body: { startDate, placeName, endDate, numberOfPeople, description, createdBy }

GET /trip/user/:userId
Purpose: Retrieve all trips created by a specific user with complete trip details.
Response: { trips: [{ _id, startDate, placeName, endDate, numberOfPeople, description, createdBy, createdAt, updatedAt }] }

GET /trip/user/:userId/:status
Purpose: Filter user trips by status (upcoming/ongoing/completed) with optional query parameters for advanced filtering.
Parameters: :status (upcoming|ongoing|completed)
Query Params: placeName, minPeople, maxPeople, startDate, endDate
Response: { trips: [...] }

GET /trip/search
Purpose: Search trips by place name with optional user filtering for global or personal trip discovery.
Query Params: userId?, q (search term)
Response: { trips: [...] }

===========================================

Suggested Places Endpoints

POST /suggested-place
Purpose: Admin-only endpoint to add curated travel destinations with images and descriptions for user inspiration.
Body: { name, description, images: [base64|urls], location, addedBy }
Authorization: Admin role required

GET /suggested-place
Purpose: Retrieve all curated travel destinations available to inspire user trip planning.
Response: { places: [{ _id, name, description, images, location, addedBy, createdAt, updatedAt }] }

===========================================
    
Itinerary Management Endpoints

POST /itinerary/create/:tripId
Purpose: Create detailed daily itinerary sections for a trip including activities, budget, and logistics information.
Body: [{ section, startDate, endDate, budget, place, activities: [], info }]
Response: { message, sections: [...] }

GET /itinerary/trip/:tripId
Purpose: Retrieve complete itinerary for a trip with aggregated statistics including total budget and activity count.
Response: { sections: [...], totalBudget, totalPlaces, totalActivities, daysCovered }

===========================================

User Profile Endpoints

GET /user/profile/:userId
Purpose: Get comprehensive user profile with personal information and categorized trips (upcoming/past).
Response: { user: {...}, upcomingTrips: [...], pastTrips: [...] }

PUT /user/update/:userId
Purpose: Update user profile information including personal details and profile picture with validation.
Body: { firstName, lastName, username, email, phoneNumber, city, country, profilePic }

===========================================

Community Endpoints

GET /community
Purpose: Retrieve all shared travel experiences and stories from the community for inspiration.
Response: { messages: [{ _id, message, author, createdAt }] }

POST /community
Purpose: Share travel experiences and stories with the community to inspire other travelers.
Body: { message, author }

===========================================

Calendar Endpoints

GET /calendar/user/:userId
Purpose: Get user's trip calendar data showing all trip dates for visual planning and conflict detection.
Response: { tripDates: [{ date, tripId, placeName }] }

===========================================

Admin Dashboard Endpoints

GET /admin/users
Purpose: Admin-only endpoint to retrieve all registered users with statistics for user management.
Authorization: Admin role required
Response: { users: [...], totalUsers, activeUsers }

GET /admin/analytics
Purpose: Admin-only endpoint to get platform analytics including popular destinations and user activity metrics.
Authorization: Admin role required
Response: { popularCities: [...], popularActivities: [...], userStats: {...} }

===========================================

Response Format

All endpoints return JSON responses with appropriate HTTP status codes:

- 200: Success
- 400: Bad Request (validation errors)
- 401: Unauthorized
- 403: Forbidden (insufficient permissions)
- 404: Not Found
- 500: Internal Server Error

===========================================

Error Handling

Error responses follow the format:

{
  "message": "Error description",
  "errors": ["Detailed validation errors"] 
}
